# 初入：NPM 包管理工具

# 1 简单了解NPM

​ NPM 全称 Node Package Manager，它是 JavaScript 的包管理工具, 并且是 Node.js 平台的默认包管理工具。

​ 因为Node.js已经集成了npm工具，接下来我们需要下载Node.js，为电脑安装Node环境。

​ 进而可以为我们的项目引入vue.js模块。

# 2 下载Node.js

中文下载地址:https://nodejs.org/zh-cn

英文下载地址:https://nodejs.org/en/download/

安装完成后，打开命令行窗口，输入node -v查看是否安装成功

# 3 初始化项目

新建一个npm-demo文件夹，打开命令提示符窗口，执行下方命令初始化项目

npm init -y   //这是以默认信息的方式初始化项目，如果要手动初始化项目，就别加-y
1

# 4 安装模块

npm install 用于安装模块，可以选择进行本地安装或者全局安装

# 4.1 本地安装

本地安装会将JS库安装到当前目录下，如果只有单个项目使用到某个模块，建议使用本地安装

打开命令提示符窗口，执行下方命令安装模块

npm install <Module Name>[@版本号]
1

比如 安装 vue 模块

npm install vue@2.6.10
1

安装后，可以在package.json 中看到我们安装的模块

package-lock:

其实用一句话来概括很简单，就是锁定安装时的包的版本号，并且需要上传到git，以保证其他人在npm install时大家的依赖能保证一致。

# 4.2 全局安装

全局安装会将JS库安装到全局目录下,如果有多个项目应用到某个模块，建议使用全局安装

打开命令提示符窗口，执行下方命令安装模块

npm install <Module Name>[@版本号] -g
1

比如安装vue模块

npm install vue@2.6.10 -g
1

查看自己的全局目录，打开命令提示符窗口，执行下方命令

npm root -g
1

# 5 生产环境模块安装

默认情况下模块都是安装到生产环境中的

生产环境模块安装方法，打开命令提示符窗口，执行下方命令

npm install <Module Name> -S
1

也可以不加 -S 因为默认情况下模块都是安装到生产环境中的

# 6 开发环境模块安装

有一些模块只是在开发的时候需要，项目上线的时候不需要，就安装到开发环境中

比如eslint模块,只是在开发阶段用来作为代码的语法校验，上线后并不需要，便可以安装到开发环境。

开发环境模块安装方法，打开命令提示符窗口，执行下方命令

npm install <Module Name> -D
1

# 7 配置淘宝镜像加速

有时候国内安装一些模块很慢，可以使用淘宝镜像加速

  1. 查看当前使用的镜像地址
npm get registry
1

2.配置淘宝镜像地址

npm config set registry https://registry.npm.taobao.org
1

3.安装下载模块

npm install <模块名>
1
  1. 还原默认镜像地址
npm config set registry https://registry.npmjs.org/
1

# 8 卸载模块

1.卸载局部模块

npm uninstall <模块名>
1

2.卸载全局模块

npm uninstall -g <模块名>
1

# 9 查看本地已安装模块

npm list
1
